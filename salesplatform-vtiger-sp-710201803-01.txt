Краткая инструкция по установке пакета обновления:
1. Перейдите в директорию, где развернута CRM (например, /var/www/vtigercrm).
2. Сделайте резервную копию файлов CRM и базы данных.
3. Скопируйте в директорию CRM архив с пакетом обновления. Для установки патча могут потребоваться права администратора.
4. Распакуйте архив:
    # tar xzf salesplatform-vtiger-sp-710201803-01.tar.gz
Архив содержит файлы с расширениями:
    patch - изменения исходного кода
    sql - изменения базы данных
    txt - описание пакета обновления и краткая инструкция.
5. Выполните команду проверки совместимости патча:
    # patch --dry-run -p 1 < salesplatform-vtiger-sp-710201803-01.patch
Вывод команды патч должен быть следующего вида:
    patching file somedir1/somefile1.php
    patching file somedir2/somefile2.php
Внимание! Если результат команды содержит строки вида Hunk #1 FAILED, не продолжайте обновление, если не являетесь разработчиком Vtiger. Среди возможных причин: не были установлены предыдущие пакеты обновления; исходный код системы изменен; версия системы не соответствует версии, для которой создан данный пакет обновления.
6. Выполните команду применения патча:
    # patch -p 1 < salesplatform-vtiger-sp-710201803-01.patch
7. Выясните имя пользователя базы данных, пароль и название базы данных Vtiger. Они заданы в файле config.inc.php (искать по тексту файла массив $dbconfig).
8. Выполните команду обновления базы данных, потребуется ввести пароль пользователя базы данных (вместо username - имя пользователя базы данных, вместо dbname - название базы данных):
    # mysql -u username -p dbname < salesplatform-vtiger-sp-710201803-01.sql
Внимание! Если результат команды содержит сообщения об ошибке выполнения SQL-команд, необходимо восстановить базу данных из резервной копии, а также откатить изменения кода:
    # patch -p 1 -R < salesplatform-vtiger-sp-710201803-01.patch
